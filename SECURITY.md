# Security Policy

## Supported Versions

`core-lab` is intended to be consumed as a shared library. The latest maintained mainline version should be considered the supported line unless a release note explicitly states otherwise.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

Instead:
- contact the maintainers through a private channel available to your team or organization
- include impact, affected packages, reproduction details, and any suggested mitigation
- allow time for triage and coordinated remediation before public disclosure

If you are unsure whether an issue is security-sensitive, treat it as private first.

## Disclosure Expectations

- We aim to acknowledge reports quickly.
- We prefer coordinated disclosure after a fix or mitigation is ready.
- Public advisories should include impacted versions, remediation steps, and upgrade guidance.
